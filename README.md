# SchoolLib — школьная библиотека

Веб-приложение для учёта книг, читателей и выдачи литературы в школьной библиотеке.

**Стек:** HTML, CSS, JavaScript (фронтенд) · Node.js + Express (сервер) · SQLite (база данных)

---

## Что нужно установить

1. **[Node.js](https://nodejs.org/)** — версия **18 LTS** или новее (рекомендуется LTS).
   - Проверка в терминале:
     ```bash
     node -v
     npm -v
     ```

2. **Git** (по желанию) — для клонирования репозитория.

Дополнительно для просмотра базы данных (необязательно):

- [DB Browser for SQLite](https://sqlitebrowser.org/) — графический просмотр `library.db`
- или CLI `sqlite3` — [sqlite.org/download](https://www.sqlite.org/download.html)

---

## Установка проекта

1. Перейдите в папку проекта:

   ```bash
   cd путь\к\SchoolLib
   ```

2. Установите зависимости:

   ```bash
   npm install
   ```

3. Если после обновления Node.js появляется ошибка про `better-sqlite3` / `NODE_MODULE_VERSION`, пересоберите модуль:

   ```bash
   npm rebuild better-sqlite3
   ```

При первом запуске автоматически создаётся файл **`library.db`** и добавляются тестовые книги и читатели.

## Запуск

### Обычный режим

```bash
npm start
```

Сервер запустится на **http://localhost:3000**

Откройте этот адрес в браузере (Chrome, Firefox, Edge).

- `/` — публичный каталог для гостей: просмотр книг и поиск.
- `/cabinet.html` — защищённый личный кабинет. Без авторизации сервер перенаправит обратно на главную.

### Открыть сайт с телефона в той же Wi‑Fi сети

После запуска `npm start` сервер выведет адреса для телефона, например:

```text
Адреса для телефона в той же Wi-Fi сети:
- http://192.168.1.25:3000
```

Откройте этот адрес на телефоне. Телефон и компьютер должны быть подключены к одной Wi‑Fi сети.

Если страница не открывается:

- разрешите Node.js доступ в Windows Firewall;
- проверьте, что телефон и компьютер в одной сети;
- попробуйте временно отключить VPN;
- убедитесь, что открываете именно адрес вида `http://192.168.x.x:3000`, а не `localhost`.

QR-коды автоматически используют Wi‑Fi адрес компьютера, поэтому их можно сканировать телефоном. Если сетевых адресов несколько, можно задать нужный вручную:

```powershell
$env:PUBLIC_URL="http://192.168.1.25:3000"
npm start
```

### Режим разработки (перезапуск при изменении файлов)

```bash
npm run dev
```

Используется встроенный флаг Node.js `--watch` (nodemon не требуется).

### Остановка сервера

В терминале нажмите **Ctrl+C**.

## Как пользоваться

Главная страница `/` содержит только публичный каталог, вход и регистрацию.
Функции пользователя, учителя, библиотекаря и администратора находятся только в `/cabinet.html`.

### Книги

- Гость может просматривать список книг и искать по названию, автору, жанру, году, ISBN и ключевым словам.
- Зарегистрированный пользователь в кабинете может добавлять книги в избранное и бронировать доступные экземпляры.
- Библиотекарь в кабинете может добавлять, редактировать и удалять книги.
  - Обязательны: название и автор.
  - «Экземпляров» — сколько физических копий есть в библиотеке.

### Бронирования и выдача

1. Ученик бронирует книгу в личном кабинете.
2. Библиотекарь подтверждает выдачу.
3. Библиотекарь подтверждает возврат.
4. История видна в личном кабинете читателя.

Сообщения об успехе и ошибках появляются в зелёной или красной полоске вверху страницы.

## Роли и тестовые аккаунты

Пароль для всех тестовых пользователей: `123456`.

| Роль | Email | Возможности |
|------|-------|-------------|
| Гость | без входа | Просмотр и поиск каталога |
| Ученик | `student@school.local` | Каталог, избранное, бронирование, отмена брони, личный кабинет |
| Учитель | `teacher@school.local` | Всё как ученик + массовое бронирование учебников на класс |
| Библиотекарь | `librarian@school.local` | Управление книгами, выдачей, возвратом, отчётами и афишей |
| Администратор | `admin@school.local` | Всё выше + пользователи, блокировки, роли, системные логи |

Новые пользователи через форму регистрации создаются с ролью `Ученик`.

### Бронирования

- Бронь создаётся на 3 дня.
- Если бронь не выдана, сервер автоматически помечает просроченные брони как `expired` и возвращает экземпляр в доступные.
- Библиотекарь переводит бронь в статус `issued`, затем при возврате — в `returned`.
- В личном кабинете отображаются активные брони, выдачи, история, избранное и текущая задолженность.

### Отчёты

Библиотекарь и администратор могут открыть отчёты:

- популярные книги;
- задолжники;
- движение книжного фонда;
- акт списания литературы.

Экспорт сделан в лёгком учебном варианте:

- **PDF** — открывается печатная HTML-версия, её можно сохранить как PDF через печать браузера;
- **Excel** — скачивается `.xls`-файл, который открывается в Excel/LibreOffice.

## База данных

| Таблица   | Назначение |
|-----------|------------|
| `books` | Книги: название, автор, жанр, год, ISBN, аннотация, ключевые слова, обложка, экземпляры |
| `users` | Пользователи, роли, пароль, класс, блокировка |
| `reservations` | Бронирования, выдачи, возвраты и история |
| `favorites` | Избранные книги пользователей |
| `events` | Афиша событий библиотеки |
| `system_logs` | История действий пользователей |
| `sessions` | Активные сессии входа |
| `readers`, `loans` | Старые учебные таблицы, оставлены для совместимости |

Схема описана в файле `db/schema.sql`.

### Просмотр через SQLite CLI

```bash
sqlite3 library.db
```

```sql
.tables
.schema
SELECT * FROM books;
.quit
```

> Рекомендуется не редактировать `library.db` вручную во время работы сервера.

### Сброс базы к начальному состоянию

1. Остановите сервер (Ctrl+C).
2. Удалите файл `library.db`.
3. Снова выполните `npm start` — БД и тестовые данные создадутся заново.

## API (для разработки)

Базовый URL: `http://localhost:3000`

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/health` | Проверка работы сервера |
| GET | `/api/auth/me` | Текущий пользователь |
| POST | `/api/auth/register` | Регистрация ученика |
| POST | `/api/auth/login` | Вход |
| POST | `/api/auth/logout` | Выход |
| GET | `/api/books` | Список книг (`?q=текст` — поиск) |
| GET | `/api/books/:id` | Одна книга |
| POST | `/api/books` | Добавить книгу (библиотекарь/админ) |
| PUT | `/api/books/:id` | Изменить книгу (библиотекарь/админ) |
| DELETE | `/api/books/:id` | Удалить книгу (библиотекарь/админ) |
| GET | `/api/reservations/mine` | Мои брони и история |
| GET | `/api/reservations` | Все брони (библиотекарь/админ) |
| POST | `/api/reservations` | Забронировать книгу |
| POST | `/api/reservations/mass` | Массовая бронь (учитель/библиотекарь/админ) |
| PATCH | `/api/reservations/:id/cancel` | Отменить бронь |
| PATCH | `/api/reservations/:id/issue` | Подтвердить выдачу (библиотекарь/админ) |
| PATCH | `/api/reservations/:id/return` | Подтвердить возврат (библиотекарь/админ) |
| GET/POST/DELETE | `/api/favorites` | Избранное |
| GET | `/api/reports/:name` | Отчёты `popular`, `debtors`, `movement`, `writeoff` |
| GET/POST/DELETE | `/api/events` | Афиша событий |
| GET/POST/PATCH | `/api/admin/users` | Пользователи (админ) |
| GET | `/api/admin/logs` | Системные логи (админ) |

Пример проверки в браузере: [http://localhost:3000/api/health](http://localhost:3000/api/health)

## Возможные проблемы

| Проблема | Решение |
|----------|---------|
| `better-sqlite3` / `NODE_MODULE_VERSION` | `npm rebuild better-sqlite3` |
| Страница не открывается | Убедитесь, что сервер запущен (`npm start`) и адрес `http://localhost:3000` |
| Порт 3000 занят | Закройте другой процесс или задайте другой порт: `set PORT=3001` (Windows) и снова `npm start` |
| Пустой список после сбоя | Проверьте консоль терминала на ошибки; при необходимости удалите `library.db` и перезапустите |
